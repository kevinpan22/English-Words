# 📡 网络访问指南

## 🔍 访问方式对比

### 方式1：本机访问（localhost）
```bash
python3 -m http.server 8000 --bind 127.0.0.1
```

- ✅ **最安全**
- ❌ 只能自己访问
- 访问地址：`http://localhost:8000`

---

### 方式2：局域网访问（推荐用于分享）
```bash
python3 -m http.server 8000 --bind 0.0.0.0
```

- ✅ 同WiFi下的设备可访问
- ⚠️ 需要在可信网络使用
- 访问地址：`http://你的IP:8000`

---

### 方式3：公网访问（不推荐）
- ❌ 需要配置路由器端口转发
- ❌ 有安全风险
- ❌ 不建议个人使用

---

## 📱 如何让别人访问（局域网）

### 步骤1：启动局域网服务器

**使用脚本（推荐）**：
```bash
cd app
./start.sh
# 选择 2（局域网访问）
```

**手动启动**：
```bash
cd app
python3 -m http.server 8000 --bind 0.0.0.0
```

### 步骤2：查看你的IP地址

**macOS**：
```bash
ipconfig getifaddr en0
# 或者
ifconfig | grep "inet "
```

**Linux**：
```bash
hostname -I
# 或者
ip addr show
```

**Windows**：
```bash
ipconfig
# 查找 IPv4 地址
```

**示例输出**：
```
192.168.1.100  ← 这就是你的局域网IP
```

### 步骤3：分享访问地址

假设你的IP是 `192.168.1.100`，端口是 `8000`

**分享给别人**：
```
http://192.168.1.100:8000
```

**别人如何访问**：
1. 连接到**同一个WiFi**（重要！）
2. 打开浏览器
3. 输入：`http://192.168.1.100:8000`

## ⚠️ 安全注意事项

### ✅ 安全的使用场景
- 家庭WiFi网络
- 办公室可信网络
- 临时演示/教学
- 团队内部分享

### ❌ 不安全的场景
- 公共WiFi（咖啡厅、机场等）
- 不可信的网络
- 长期暴露在网络上

### 🛡️ 安全建议

1. **使用时才开启**
   ```bash
   # 用完后按 Ctrl+C 停止服务器
   ```

2. **检查防火墙**
   - macOS: 系统偏好设置 → 安全性与隐私 → 防火墙
   - Windows: 控制面板 → Windows Defender 防火墙

3. **仅在可信网络使用**
   - 确保WiFi有密码保护
   - 确认连接的设备都是可信的

## 🔧 常见问题

### Q1: 别人无法访问怎么办？

**检查清单**：
- [ ] 确认连接同一WiFi
- [ ] 确认服务器正在运行
- [ ] 确认IP地址正确
- [ ] 检查防火墙设置
- [ ] 尝试 ping 你的IP：`ping 192.168.1.100`

### Q2: 如何查看谁在访问？

服务器会显示访问日志：
```
192.168.1.101 - - [10/Apr/2026 15:30:45] "GET /index.html HTTP/1.1" 200 -
```

### Q3: 如何限制访问？

Python 简单服务器无法限制访问。如需限制：
- 使用时才开启服务器
- 用完后立即停止（Ctrl+C）
- 不要在公共网络使用

### Q4: 可以用手机号流量访问吗？

❌ **不可以**
- 局域网访问仅限同一WiFi
- 手机流量是独立网络
- 如需远程访问，考虑：
  - 云服务器部署
  - GitHub Pages 托管
  - Vercel/Netlify 等服务

## 🚀 替代方案（云端部署）

如果想让**任何人**随时访问（不限同一WiFi）：

### 方案1：GitHub Pages（免费）
```bash
# 1. 上传到 GitHub
git init
git add .
git commit -m "deploy"
git push origin main

# 2. 启用 GitHub Pages
# Settings → Pages → Source: main branch
```
访问地址：`https://你的用户名.github.io/仓库名`

### 方案2：Vercel（免费）
```bash
# 1. 安装 Vercel CLI
npm i -g vercel

# 2. 部署
cd app
vercel
```

### 方案3：Netlify（免费）
拖拽 `app` 文件夹到 https://app.netlify.com/drop
